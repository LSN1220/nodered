# 07-db-schema-iot

这是第七组可运行样例，目标是学习物联网场景的数据库设计。

物联网数据和管理后台数据很不一样。后台更关注用户、角色、权限；物联网更关注设备、状态、事件、告警和规则。

## 学习目标

- 理解设备类型和设备实例的关系
- 理解设备当前状态表
- 理解事件数据表
- 理解告警规则和告警记录
- 理解为什么状态数据和历史事件数据要分开

## 运行方式

```powershell
npm install
npm run seed
npm start
```

## 核心表

- `device_types`：设备类型
- `devices`：设备实例
- `device_status`：设备当前状态
- `device_events`：设备事件历史
- `alert_rules`：告警规则
- `alerts`：告警记录

## 推荐练习

- 新增一个湿度传感器设备类型
- 增加一条新的温度事件
- 修改告警阈值后重新运行 demo
